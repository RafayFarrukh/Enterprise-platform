## ğŸ¯ **Key Design Principles**

1. **Single Source of Truth**: Auth Service is the only source for authentication
2. **JWT Tokens**: All services validate JWT from Auth Service
3. **RBAC**: Role-Based Access Control with permissions
4. **Atomic Transactions**: Wallet operations use database transactions
5. **Audit Trail**: All wallet transactions are logged
6. **Idempotency**: Payment operations are idempotent (can be retried safely)
7. **Service Isolation**: Each service has its own database
8. **API Gateway**: Single entry point for all requests

---

## ğŸ“Š **Database Distribution**

- **enterprise_sso** (PostgreSQL): Users, Agencies, Roles, Permissions, Sessions
- **enterprise_users** (PostgreSQL): User Profiles, Wallets, Wallet Transactions
- **enterprise_agencies** (PostgreSQL): Agency Profiles, Business Data
- **enterprise_orders** (PostgreSQL): Orders, Order Items, Gateway Transactions
- **enterprise_chat** (MongoDB): Conversations, Messages
- **Redis**: Cache, Sessions, Rate Limiting
